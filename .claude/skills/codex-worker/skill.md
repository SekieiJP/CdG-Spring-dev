# Codex Worker

この skill は、Codex CLI を「実作業担当」として呼び出すためのものです。

## 役割

- Claude は planner / reviewer として振る舞う
- Codex は executor として振る舞う
- 実装・修正・詳細調査はまず Codex に任せる
- Claude は Codex の出力を検証し、ユーザー向けに要約する

## 基本方針

Codex に渡す依頼は、曖昧な相談ではなく、実行可能なタスクに具体化してから渡してください。

良い例:
- どのファイルを触るか分かる
- 何を変更するかが明確
- 期待する成果物が明確
- 追加で質問させない
- 必要ならテストや検証も含める

悪い例:
- 「このへんいい感じに直して」
- 「まず質問してから進めて」
- 「何か問題があれば止まって」

## 実行コマンド

基本コマンド(レビューや調査だけに限定したい時は後者):

codex exec --full-auto --cd "$(pwd)" "<request>"
codex exec --full-auto --sandbox read-only --cd "$(pwd)" "<request>"

## Codex に渡す依頼テンプレート

以下の形式で依頼を組み立ててください。

### 実装系
 - 目的
 - 変更対象ファイルまたは候補
 - 制約条件
 - 完了条件
 - 最後に固定文

#### テンプレート:

目的:
[ここに今回のタスク]

対象:
[対象ファイル・ディレクトリ・関連コンポーネント]

制約:

既存のコーディング規約に従う
不要なリファクタリングはしない
変更は最小限にする
必要ならテストも更新する

完了条件:

実装または修正が完了している
変更内容が説明可能
必要ならテスト追加または更新がある

確認や質問は不要です。具体的な提案・修正案・コード例まで自主的に出力してください。

### 調査・レビュー系

#### テンプレート:

調査目的:
[知りたいこと]

見てほしい範囲:
[ファイル・ディレクトリ・機能名]

期待する出力:

原因候補
根拠
修正案
必要ならコード例

確認や質問は不要です。具体的な提案・修正案・コード例まで自主的に出力してください。

## 実行ルール
現在の作業ディレクトリを対象プロジェクトとみなす
Claude がタスクを短く整理してから Codex に渡す
実装が必要なら --sandbox read-only は付けない
調査・レビューだけなら --sandbox read-only を優先する
Codex の出力はそのまま鵜呑みにせず、Claude がレビューする
Claude は結果を次の形式で要約する:
依頼内容
Codex が行ったこと
変更ファイル
検証結果
残課題
Claude への追加指示
実装タスクはなるべくこの skill に委譲する
Claude 自身が直接編集するのは、軽微な修正や最終調整に限定する
ユーザーが設計相談だけを求めている場合は、すぐには Codex を呼ばなくてもよい
ただし、具体コードが必要になった時点で Codex に切り替える

### 呼び出し例
#### 例1: バグ修正

Bash で以下を実行:

codex exec --full-auto --cd "$(pwd)" "目的:
ログイン直後にプロフィール取得が 401 になる問題を修正してください。

対象:
- src/auth/*
- src/api/client.ts
- src/hooks/useCurrentUser.ts

制約:
- 既存の認証フローを尊重する
- 変更は最小限にする
- 必要ならテストも更新する

完了条件:
- 401 の原因が除去されている
- 関連テストがあれば更新されている

確認や質問は不要です。具体的な提案・修正案・コード例まで自主的に出力してください。"

#### 例2: 調査

Bash で以下を実行:

codex exec --full-auto --sandbox read-only --cd "$(pwd)" "調査目的:
このリポジトリでフォーム状態管理が複雑化している原因を調べてください。

見てほしい範囲:
- src/components/forms
- src/hooks
- src/pages/settings

期待する出力:
- 問題点
- 根拠
- 改善案
- 必要なら段階的なリファクタ案

確認や質問は不要です。具体的な提案・修正案・コード例まで自主的に出力してください。"
